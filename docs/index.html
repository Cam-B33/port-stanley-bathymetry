Methodology

Overview

This project uses multispectral satellite imagery from the European Space Agency's Sentinel-2 constellation to map nearshore bathymetry. Different wavelengths of light penetrate the watercolumn at different depths, by taking the ratio from two different bands of light, a relative model of underwater topography can be created.

Band Ratio Method

Relative depth Z is approximated as:

Z ≈ ln(Rgreen) / ln(Rred)

where R denotes reflectance in each spectral band.

Physical Principles

Water absorbs light differently based on wavelength. Red light (620-670 nm) is absorbed more rapidly than green light (530-590 nm). In shallow water, bottom reflectance contributes to the upwelling radiance measured by the satellite sensor.

The ratio of logarithmic reflectances normalizes for variations in water clarity and illumination while remaining sensitive to depth variations in the 0-10m range typical of nearshore environments.

Data Processing Pipeline

  1. Acquisition: Sentinel-2 Level-2A (atmospherically corrected) imagery at 10m spatial resolution
  2. Band Selection: Blue (B02), Green (B03), and Red (B04) bands
  3. Ratio Calculation: Log-ratio bathymetry algorithm
  4. Clipping: Extract area of interest (aoi) around Port Stanley's Main Beach
  5. Visualization: Generate contoured bathymetric maps displaying the relative underwater topography

Limitations

This method provides relative rather than absolute depth measurements. Accuracy decreases with increasing turbidity, wave action, and sunlight reflectance. Significant cloud cover can prevent data analysis. Update frequency is affected by the satellite orbit (5 days for Sentinel-2 constellation), and ground conditions that impact data collection.

Limitations

The model assumes steady-state flow conditions and does not account for temporal variations in wave action, wind, or current velocity. Actual rip current activity depends on these factors in addition to bathymetry.

Open Source Implementation

The complete processing pipeline is available as an open-source CLI program, available on github. The program is designed to be a reproducible analysis pipeline that is easily adaptable to other nearshore environments.

Installation

# Clone repository
git clone https://github.com/Cam-B33/port-stanley-bathymetry.git
cd port-stanley-bathymetry

# Install dependencies
pip install -r requirements.txt

Basic Usage

# Process a Sentinel-2 scene
python sentinel_bathymetry.py process \
    --scene /path/to/S2_SAFE_folder \
    --aoi aoi.geojson \
    --output port_stanley_2024_06_10

Automated Processing

The system can automatically download and process new imagery every 10 days:

# Download and process recent imagery
python sentinel_bathymetry.py download-and-process \
    --aoi aoi.geojson \
    --start-date 20240601 \
    --end-date 20240610 \
    --username your_copernicus_username \
    --password your_password
